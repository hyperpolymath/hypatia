== Overview

Hypatia is the neurosymbolic CI/CD intelligence layer for the hyperpolymath ecosystem. It coordinates the gitbot-fleet (rhodibot, echidnabot, sustainabot, glambot, seambot, finishbot) via a safety triangle pipeline, with 5 neural networks, VQL query layer, Bayesian confidence updating, and Logtalk rules for pattern detection.

== Current State (2026-03-29)

=== Operational Metrics

* 302 repos scanned, 3385 weak points tracked
* 1635 dispatched actions (600 auto-execute, 667 review, 368 report)
* 16671 outcomes recorded (99% success rate, Bayesian confidence updating)
* 46 recipes (0 uncovered categories), 20 OpenSSF Scorecard recipes, 5 RSR compliance rules
* 954 canonical patterns (PA001-PA020+)
* 295 summaries auto-generated for verisimdb-data repos

=== Architecture (OPERATIONAL)

[cols="1,3"]
|===
| Layer | Details

| Elixir pipeline
| 12+ core modules: PatternAnalyzer, VerisimdbConnector, PatternRegistry, RecipeMatcher, TriangleRouter, FleetDispatcher, DispatchManifest, OutcomeTracker, RecipeGenerator, ScorecardIngestor, LearningScheduler, SelfDiagnostics

| Neural subsystem
| 5 networks + coordinator GenServer: Graph of Trust (PageRank), Mixture of Experts (7 domains), Liquid State Machine (temporal anomaly), Echo State Network (trajectory forecast), Radial Neural Network (similarity/novelty)

| VQL query layer
| Built-in parser, file executor, query cache, cross-repo analytics

| Safety systems
| Rate limiter (per-bot 50/min, global 200/min, burst 10/5s), quarantine (3 levels: soft/hard/permanent), batch rollback

| OTP supervision
| 8 GenServers: VQL, RateLimiter, Quarantine, Learning, Diagnostics, Neural, Kin, Application

| Idris2 ABI
| Types, GraphQL, gRPC, REST endpoint definitions with dependent type proofs

| Zig FFI
| 7 exported C ABI functions (health, scan, dispatch, outcome, learning, confidence, strategy)

| Rust workspace
| adapters, cli, data, fixer, integration crates

| Logtalk rules
| Error catalog, pattern detection, RSR compliance rules

| Data layer
| verisimdb-data (git-backed flat-file store), neural state persistence, JSONL outcomes
|===

=== Safety Triangle Pipeline (OPERATIONAL)

```
panic-attack assail (scan repos)
        | JSON results
verisimdb-data (git-backed flat-file store, 302 repos)
        | VQL queries
Elixir pipeline:
  PatternRegistry → TriangleRouter (Eliminate > Substitute > Control)
        |
  FleetDispatcher (confidence-gated, circuit breaker)
        |
  dispatch-runner.sh (gitbot-fleet)
    ├── auto_execute (>=0.95): robot-repo-automaton
    ├── review (0.85-0.94): rhodibot creates PR
    └── report_only (<0.85): sustainabot advisory
        |
  OutcomeTracker (Bayesian Beta-distribution, re-scan verification)
```

== Known Gaps

1. **VQL federation local-only:** FileExecutor handles FEDERATION queries against local files, not multi-store
2. **verisim-api not deployed:** VeriSimDB Rust core not running — graph/vector/temporal via flat files only
3. **One-sided training data:** 99%+ outcomes are "success" — needs failure data for balanced learning
4. **Fix script coverage:** 310/600 auto-execute entries have null fix_script
5. **Containerfiles:** SWI-Prolog and Haskell still use non-Chainguard base images
6. **Ada TUI not integrated:** Compiles but not wired into Elixir supervision tree
7. **Neural state persistence:** State dir exists but coordinator has not persisted to disk yet
