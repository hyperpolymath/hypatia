= Hypatia -- OMG MOF (M2) Metamodel
:toc: left
:toclevels: 3
:sectnums:
:icons: font

[NOTE]
====
*Canonical, as-built model.* This is an OMG MOF *M2 metamodel* of Hypatia's core
domain. It reflects the *as-built* system (flat files, no ArangoDB/snif/NIF).
Target-state boundary elements are tagged *TARGET (pending ruling #273)* and are
*not* implemented. Legacy Mermaid diagrams under `docs/diagrams/*.md` are
superseded -- see `README.adoc`.
====

== 1. MOF Layering

[cols="1,1,3",options="header"]
|===
| Layer | Name | Content here

| M3 | MOF (meta-metamodel) | OMG MOF (`Class`, `Property`, `Association`, `Generalization`) -- assumed, not redefined.
| M2 | Hypatia metamodel | The metaclasses below (Finding, Scan, Recipe, ...).
| M1 | Model | A concrete configuration: a particular scan, its findings, the recipe registry.
| M0 | Instances | Runtime data: a row in `data/verisim/scans/<repo>.json`, a line in `pending.jsonl`.
|===

== 3. Meta-Associations

[cols="1,1,1,3",options="header"]
|===
| Source | Multiplicity | Target | Semantics

| `Scan` | 1 -> 0..* | `Finding` | A scan yields zero or more findings.
| `Finding` | 0..* -> 0..1 | `Pattern` | A finding may match a known pattern (classification).
| `Pattern` | 1 -> 0..* | `Recipe` | A pattern has candidate remediation recipes.
| `Recipe` + `Pattern` | 1 + 1 -> 1 | `RoutedAction` | Classification produces a routed action.
| `RoutedAction` | 0..* -> 1 | `DispatchManifest` | Routed actions are batched into a manifest (`pending.jsonl`).
| `DispatchManifest` | 1 -> 1..* | `Bot` | Manifest entries are dispatched to fleet bots.
| `Bot` | 1 -> 0..* | `Outcome` | Bot remediation produces outcomes.
| `Outcome` | 0..* -> 1 | `NeuralOrgan`(via Coordinator) | Outcomes feed `Coordinator.record_outcome` (`lib/fleet_dispatcher.ex:691`).
| `NeuralOrgan` | 8 -(blackboard)- 1 | `RoutedAction` | Organs (6 phases) advise/adjust routing (`lib/pattern_analyzer.ex:258-308`).
| `OctadEntity` | 1 -> 1 | {Scan,Pattern,Recipe,Outcome,DispatchManifest} | Each domain class is persisted as one OctadEntity store kind (flat file).
|===

== 4. Constraints (OCL-style, informal)

* `inv as_built_store`: every `OctadEntity.physicalForm` is one of
  `{jsonDir, singleJson, jsonlDir}` -- *no database-backed form exists*
  (`lib/vcl/file_executor.ex:99-104`).
* `inv neural_advisory`: a `RoutedAction.kind` is never *solely* determined by a
  `NeuralOrgan`; symbolic `Recipe` is the primary determinant; neural sets
  `neuralAdjusted` (`lib/pattern_analyzer.ex:258-308`, `lib/kin/gate.ex:402-413`).
* `inv findings_write_bypasses_vcl`: `DispatchManifest` and the fix journal are
  written as raw JSONL, *not* via the VCL query path
  (`lib/dispatch_manifest.ex:33,57-58`, `lib/fleet_dispatcher.ex:707`).
* `inv proof_resolver_unbound`: no metaclass references the (vestigial)
  `proof_resolver` HTTP seam; it has zero callers.
