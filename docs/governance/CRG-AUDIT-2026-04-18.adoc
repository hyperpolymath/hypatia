== Scope and Standard

Evaluates the `hypatia` repository (neurosymbolic-intelligence rule engine for
the hyperpolymath estate) against *Component Readiness Grades v2.0 (STRICT)*,
2026-03-30 revision. Paired with the boj-server + gossamer + academic-workflow-suite
audits written the same day.

- Standard: `standards/component-readiness-grades/COMPONENT-READINESS-GRADES.md`
- Criteria boilerplate: `rsr-template-repo/docs/governance/CRG-CRITERIA.adoc`
- Audit template: `rsr-template-repo/docs/governance/CRG-AUDIT-TEMPLATE.adoc`
- Self-declared grade (prior art):
  * `.machine_readable/6a2/STATE.a2ml` has *no* `crg-grade` / `[crg]` entry at all.
    Repo declares `completion-percentage = 78`, `maturity = "alpha"`,
    `status = "active"` — no CRG letter claimed in STATE.a2ml as of today.
  * `READINESS.md` (v1.0 schema, 2026-04-04) asserts *overall Grade B*, with
    three of six components (`pattern-analyzer`, `recipe-matcher`,
    `hypatia-scan.yml`) graded `B` and three (`triangle-router`,
    `fleet-dispatcher`, `neural-coordinator`) graded `C`.
- Audit date: 2026-04-18
- Auditor: Jonathan D.A. Jewell

Schema-drift note: STATE.a2ml omits any CRG key entirely. HYP-S005 (flat
`crg-grade = …`) and the gossamer-style sectioned (`[crg] / grade = …`) both
fail to detect a grade here. The canonical document the audit must refute is
therefore `READINESS.md`, not STATE.a2ml. This is itself a governance defect:
the authoritative machine-readable state does not carry a CRG field.

The audit grades the repo *as-is today*, not aspirationally.

== Verdict

*Current CRG grade: **D (Alpha), with one blocker to C and four supporting gaps**.*

Self-declared grade: *B* (from `READINESS.md`; STATE.a2ml declares no grade).
Audited grade: *D*. **This is an overclaim of two grades**, consistent with
the academic-workflow-suite finding and different in character from the
gossamer finding (1-grade overclaim): hypatia's problem is *not* structural
debt (structurally it is the cleanest repo audited this week) — it is the
combination of (a) unresolved red CI, (b) per-directory annotation floor
failure, and (c) confusing language-diversity within one estate with
external-target diversity.

Recommended action: populate `.machine_readable/6a2/STATE.a2ml` with a new
`[crg]` section (sectioned form, matching gossamer's convention):

[source]
----
[crg]
grade = "D"
grade-previous = "B (self-declared in READINESS.md, refuted by this audit)"
grade-date = "2026-04-18"
next-target = "C"
notes = """
CRG v2.0 STRICT audit 2026-04-18: 33 RSR workflows, 697 tests, 1,128 LOC
Idris2 ABI with 0 believe_me uses, clean container+deno+justfile posture,
Logtalk purge complete. Blocker to C: 124 pre-existing test failures
recorded 2026-04-05 not yet closed (CI not green clause). Per-subtree
README coverage 1/22. External-target trail absent; language diversity
within estate is not v2.0 B evidence.
"""
----

and retire the *B* claim in `READINESS.md` with an audit-citing note.

== Detector-Rule Drift (cross-cutting)

Hypatia's STATE.a2ml has *no* CRG key at all. Neither HYP-S005 (flat
`crg-grade = …`) nor the gossamer-style sectioned (`[crg] / grade = …`)
detector pattern matches this repo — because the field is absent, not
drifted. The correct detector behaviour for "CRG key absent from STATE.a2ml"
is a distinct finding from "CRG key present but schema drifts":

- *Absent*: repo makes no machine-readable CRG claim — raise as
  `HYP-S005-ABSENT` (advisory, not a contradiction).
- *Drifted-schema*: flat vs sectioned — raise as `HYP-S005-SCHEMA`
  (cross-cutting with gossamer finding).
- *Contradiction*: STATE.a2ml CRG disagrees with `READINESS.md` CRG —
  raise as `HYP-S005-CONTRADICTION` (the academic-workflow-suite + hypatia
  failure mode).

This third case is what hypatia exhibits today: STATE.a2ml is silent,
`READINESS.md` claims B, audit finds D. Capture as follow-up against
`developer-ecosystem/standards/hypatia-rules/`. Note: hypatia itself
currently has no rule to detect its own CRG contradiction — a worthwhile
dogfood target.
