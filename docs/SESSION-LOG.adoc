== Satellite Repository Protocol

When a repo is a satellite of another:

=== For Satellite Repos

1. Document in `ECOSYSTEM.scm`:
   - `(parent-project "git-dispatcher")`
   - `(relationship satellite)`
   - `(update-protocol ...)`

2. After completing work, commit must include:
   - Update to STATE.scm with changes
   - Note about parent repo needing notification

3. Push to satellite triggers:
   - Notification to parent repo (via webhook or PR)
   - Parent documentation may need update

=== For Central/Parent Repos

1. Document in `ECOSYSTEM.scm`:
   - `(satellites (list "cicd-hyper-a" "other-satellite"))`
   - Track which satellites exist

2. When satellite is updated:
   - Review satellite changes
   - Update own documentation if needed
   - Acknowledge in 6scm files

=== Cryptographic Standards (Audited 2026-01-04)

See `CRYPTO-STANDARD.scm` for full specification.

**Principle:** Post-quantum ready, proven algorithms, flat prime distributions

|===
|Category |Required Algorithm |NIST Standard |Status

|Password hashing |argon2id (64 MiB, 3 iter, 4 threads) |- |Enforced
|General hashing |blake3, shake3-256 |- |Enforced
|Post-quantum signatures |dilithium5 |ML-DSA-87 (FIPS 204) |Enforced
|Classical signatures |ed448 (x25519 for SSH) |- |Enforced
|Post-quantum key exchange |kyber-1024 |ML-KEM-1024 (FIPS 203) |Enforced
|Classical key exchange |x448 (x25519 acceptable) |- |Enforced
|Symmetric encryption |chacha20-poly1305, xchacha20-poly1305 |- |Enforced
|===

**Banned algorithms:** md5, sha1, sha256 (use blake3), bcrypt, scrypt, pbkdf2, rsa (except legacy interop), ecdsa, ecdh-p256/p384, aes-128-*, ecb-mode
