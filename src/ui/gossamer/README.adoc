= Hypatia GUI (Gossamer + Ephapax)
:toc:

// SPDX-License-Identifier: PMPL-1.0-or-later

Replacement for the earlier AffineScript GUI at `src/ui/tea/`. Per
issue #177 the GUI is migrated to Gossamer (UI framework) on top of
Ephapax (dyadic-typed WASM language), with PanLL hosting the panel
and Burble available as an optional session layer (deferred — see
`BURBLE-DEFERRAL.md`).

== Files

=== Source

* `hypatia_gui.eph` — TEA loop (Model / Msg / update / view / subs).
  Port of `hypatia_gui.affine`; behaviour identical.
* `bridge.eph` — Gossamer host integration (window + IPC handles).
  JSON wire-format decoder for incoming `Msg` frames.
* `loader.js` — browser-side loader for the Gossamer-compiled wasm.
  Defines the JS `Msg` constructors that pair with `decode_msg` in
  `bridge.eph`.

=== PanLL lifecycle artefacts

Four files map to PanLL's four lifecycle tools (minter, provisioner,
configurator, harness):

* `panll.minter.toml` — contract fields the minter writes upstream
  at `panll/contracts/hypatia.toml`. Already exists upstream.
* `panll.provisioner.toml` — resources the provisioner stands up
  (isolation tier, ports, filesystem paths, container image).
* `panll.configurator.cue` — runtime config conforming to
  `panll/schemas/config.cue` `#Config`.
* `panll.harness.toml` — operational manifest validated against
  `panll/schema/panll-harness-v2.schema.json`.

Syntactic sanity check on `.eph` sources (2026-04-22): both files
pass by-hand validation against the Ephapax v2 grammar (bracket
balance, `match`/`end` pairing, linear `let!` consumption, pattern
constructor alignment). Type-check pending toolchain availability.

== Local build (once Ephapax toolchain is in)

[source,bash]
----
# Compile the TEA core + bridge into a single wasm module.
eph build --target wasm32 \
  src/ui/gossamer/bridge.eph \
  -o src/ui/public/assets/wasm/hypatia_gui.wasm

# Sanity-check the wasm module.
wasm-validate src/ui/public/assets/wasm/hypatia_gui.wasm

# Serve the UI locally.
cd src/ui/public && python3 -m http.server 8000
# open http://localhost:8000
----

== Why Ephapax over AffineScript

Summary: Gossamer is already built on Ephapax, Ephapax is dyadic
(affine + linear, not affine-only), and the Hypatia GUI has linear
resources (IPC channel, window handle) that benefit from linear
discipline. Full reasoning in issue #177.
