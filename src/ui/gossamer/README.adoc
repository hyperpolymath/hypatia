= Hypatia GUI (Gossamer + Ephapax)
:toc:

// SPDX-License-Identifier: PMPL-1.0-or-later

Replacement for the earlier AffineScript GUI at `src/ui/tea/`. Per
issue #177 the GUI is migrated to Gossamer (UI framework) on top of
Ephapax (dyadic-typed WASM language), with PanLL hosting the panel
and Burble available as an optional session layer (deferred — see
`BURBLE-DEFERRAL.md`).

== Files

=== Source

* `hypatia_gui.eph` — TEA loop (Model / Msg / update / view / subs).
  Port of `hypatia_gui.affine`; behaviour identical.
* `bridge.eph` — Gossamer host integration (window + IPC handles).
  JSON wire-format decoder for incoming `Msg` frames.
* `loader.js` — browser-side loader for the Gossamer-compiled wasm.
  Defines the JS `Msg` constructors that pair with `decode_msg` in
  `bridge.eph`.

=== Lifecycle artefacts (minter / provisioner / configurator / harness)

The four artefacts in this directory follow a pattern that is general
across the ecosystem — not specific to PanLL panels:

* PanLL plugins and panels (Hypatia ships as a panel)
* BoJ cartridges (`hyperpolymath/boj-server`)

Source-of-truth ordering for the cross-tool link semantics:

* `hyperpolymath/standards` — `groove-protocol/spec/` is the canonical
  spec for how groove-aware systems discover, type-check, and compose
  with each other (Burble, Gossamer, VeriSimDB are reference
  implementations).
* `hyperpolymath/panll` — per-PanLL guidance:
  `schema/panll-harness-v2.schema.json`, `schemas/config.cue`,
  and `contracts/<panel-id>.toml`.
* `hyperpolymath/gossamer` — per-Gossamer guidance for host integration;
  `src/core/Bridge.eph` is the canonical pattern.
* `hyperpolymath/boj-server` — per-BoJ guidance for cartridge
  construction.

PanLL itself is a single tool that branches between a *plugin* path
and a *panel* path. Hypatia takes the panel path. The architectural
relationship to the Binary Star layout is by **flow** and **viewing
perspective** (third-position, or from-L, or from-N, mediated by
which Pane-A is viewed through), not by membership in a single pane.
The four panes per `hyperpolymath/panll/README.adoc:99-110`:

* Pane-A — Ambient (persistent context / environmental support)
* Pane-L — Logic (role: Symbolic; strict logic, types, constraints)
* Pane-N — Neural (streaming inference manifold, OODA traces)
* Pane-W — World (task barycentre; results, actions, artefacts)

Hypatia's `clade = "neurosym/ci-cd-intelligence"` records topical
taxonomy. The architectural relationship is declared in
`panll.harness.toml`'s `[binary_star]` section: Hypatia spans L
(rules engine, safety triangle, error catalog) and N (5 neural
networks: Graph of Trust, Mixture of Experts, LSM, ESN, RBF), flow
runs `L → N → W`, outputs surface in W, perspective is
*meta-observer*. The `binary_star` schema fields are defined in
`hyperpolymath/panll`'s `panll-harness/v2` schema (see
https://github.com/hyperpolymath/panll/pull/26[panll#26]).

The four files in this directory:

* `panll.minter.toml` — local mirror. Canonical contract upstream is
  `hyperpolymath/panll:contracts/hypatia.toml` (verified present
  2026-04-30).
* `panll.provisioner.toml` — resources the provisioner stands up
  (isolation tier, ports, filesystem paths, container image).
* `panll.configurator.cue` — runtime config; conforms to
  `panll/schemas/config.cue` `#Config`.
* `panll.harness.toml` — operational manifest; validates against
  `panll/schema/panll-harness-v2.schema.json` (CI-enforced).

Syntactic sanity check on `.eph` sources (2026-04-22): both files
pass by-hand validation against the Ephapax v2 grammar (bracket
balance, `match`/`end` pairing, linear `let!` consumption, pattern
constructor alignment). Type-check pending toolchain availability.

== Migration status

|===
| Item | Status

| AffineScript → Ephapax port
| ✅ Complete (`hypatia_gui.eph`, `bridge.eph`)

| Msg wire-format decoder
| ✅ Implemented against JSON. Depends on three JSON-helper externs
  (`bytes_to_string`, `json_parse_tag`, `json_parse_int_field`) that
  will collapse into one call once Ephapax publishes a JSON stdlib
  module.

| PanLL harness manifest
| ✅ Validated against `panll-harness/v2`. Four lifecycle artefacts
  present.

| Burble session integration
| ⏸  Deferred. See `BURBLE-DEFERRAL.md`.

| Loader smoke harness (Node)
| ✅ `test/gossamer/loader_smoke.test.mjs` — locks the
  `extern "gossamer"` import surface, exercises multi-channel IPC,
  buffered-send-on-`onopen`, and instantiates a hand-built minimal
  `.wasm` against `loader.js`. Runs in the `Build Gossamer GUI`
  workflow on every PR touching `src/ui/gossamer/**`. No upstream
  Ephapax dependency.

| Ephapax toolchain build (CI)
| ✅ `.github/workflows/build-gossamer-gui.yml` clones
  `hyperpolymath/ephapax` at v1.0.0 (commit `c14c3e92`) and runs
  `cargo build --release --workspace`. Result is cached per-SHA, so
  subsequent runs are seconds.

| `.eph` → `.wasm` compilation (CI)
| ⏸  Gated on the v2 grammar in `ephapax-parser` upstream
  (hyperpolymath/ephapax#36). Ephapax v1.0.0 already ships the
  `ephapax compile <file> -o <out>` subcommand, but its parser only
  accepts top-level `fn` and `type` declarations. `bridge.eph` uses
  v2 constructs (`module`, `import`, `extern "gossamer"`, `data`,
  `match`, `let!`) that the v1.0.0 parser rejects with a syntax
  error — the missing piece is the parser, not the CLI wrapper.
  The workflow probes for `ephapax compile`, `ephapax compile-eph`,
  `ephapax compile-affine`, and `just compile-affine`; whichever
  starts accepting `bridge.eph` first wins. On successful compile,
  the workflow runs `wasm-validate` (wabt) on the produced
  `hypatia_gui.wasm` before uploading the artifact.
|===

== Local build

Prerequisites: Rust 1.83+ with `wasm32-unknown-unknown` target, `just`,
`git`. Clone and build Ephapax once:

[source,bash]
----
git clone https://github.com/hyperpolymath/ephapax /tmp/ephapax
cd /tmp/ephapax
git checkout v1.0.0    # or: c14c3e92a9a6bfc1074ea9d97a3ce4b5fae37501
just doctor            # optional — reports missing tools
cargo build --release --workspace
----

Once an upstream `.eph` → `.wasm` CLI subcommand lands, the Hypatia
side call is:

[source,bash]
----
# Example — exact subcommand name depends on which of compile-affine /
# compile-eph ships first. Check `ephapax-cli --help` and `just --list`.
/tmp/ephapax/target/release/ephapax-cli compile-eph \
  src/ui/gossamer/bridge.eph \
  -o src/ui/public/assets/wasm/hypatia_gui.wasm

wasm-validate src/ui/public/assets/wasm/hypatia_gui.wasm

cd src/ui/public && python3 -m http.server 8000
# open http://localhost:8000
----

== Manual browser smoke test

Until the end-to-end compile path lands upstream and a wasm artifact is
produced by CI, `index.html` loading + console-error-free behaviour is a
manual step:

. Start the Hypatia OTP listener (Bandit + Plug.Router): `iex -S mix`
  for an interactive shell, or `mix run --no-halt` for a headless run.
  This serves on `:9090` by default; override via the `:hypatia,
  :http_port` application env. (The repo isn't a Phoenix app, so
  `mix phx.server` doesn't apply.)
. Place a compiled `hypatia_gui.wasm` at
  `src/ui/public/assets/wasm/hypatia_gui.wasm` (download the artifact
  from the `Build Gossamer GUI` workflow once it is producing one).
. From `src/ui/public/`, run `python3 -m http.server 8000`.
. Open `http://localhost:8000` and confirm: panel renders, no console
  errors, clicking each department button updates `#app`'s text.

The loader (`loader.js`) supplies a browser-side `gossamer` host import
namespace that mirrors the `extern "gossamer"` block in `bridge.eph`.
The string ABI (`{ptr, len}` with leading i32 length prefix) is
documented in `loader.js`; once Ephapax pins the wasm-level packing
this is the single point of change.

IPC support is multi-channel (one `WebSocket` per `ipc_open` handle,
keyed in a `Map`), and `ipc_send` payloads issued before the socket
reaches `OPEN` are buffered and flushed on `onopen` rather than
silently dropped. A WS implementation can be injected via the second
arg to `makeGossamerHost(memory, { WebSocket })` for headless tests.

== Automated smoke tests

`test/gossamer/loader_smoke.test.mjs` runs under `node --test` (no
extra deps) and locks the loader contract that the manual browser
smoke step exercises end-to-end. Run locally with:

[source,bash]
----
node --test test/gossamer/loader_smoke.test.mjs
----

The harness builds a minimal valid `.wasm` from raw bytes (no Ephapax
toolchain needed), instantiates it through `loader.js`'s gossamer
import object, and verifies the `{ptr, i32-len}` string ABI
round-trips through real wasm memory. CI runs it on every PR that
touches `src/ui/gossamer/**`.

== Why Ephapax over AffineScript

Summary: Gossamer is already built on Ephapax, Ephapax is dyadic
(affine + linear, not affine-only), and the Hypatia GUI has linear
resources (IPC channel, window handle) that benefit from linear
discipline. Full reasoning in issue #177.
