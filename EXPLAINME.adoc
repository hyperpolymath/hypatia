== Key Claims From README

[quote, README section "Core Philosophy"]
____
"Dumb rules from smart learning"

The system learns complex patterns through neural analysis, then *distills* them into simple, fast-acting declarative rules that can be pre-emptively injected into new repositories, applied as cures to existing repositories, executed without ML inference overhead, and formally verified for correctness.
____

*Evidence:* The learning pipeline (CLAUDE.md line 77-101) demonstrates this cycle:
1. **Learning** — 5 neural networks (Graph of Trust, Mixture of Experts, Liquid State Machine, Echo State Network, Radial Neural Network) analyze verisimdb-data
2. **Distillation** — PatternRegistry deduplicates findings into 46 canonical recipes (PA001-PA046 + 20 OpenSSF Scorecard recipes)
3. **Symbolic** — Recipes execute as fast Logtalk rules (no ML at runtime)
4. **Verification** — Idris2 ABI proofs formally verify recipe correctness

This is proven by the metrics: 1635 dispatched actions (600 auto-execute with 95%+ confidence, 667 review, 368 report), 16,671 outcomes recorded (99% success rate), 46 recipes zero-uncovered (every category has a recipe).

*Caveat:* The "dumb rules" are fast but not universally applicable — categories below 0.85 confidence force review (not auto-execute), and below 0.50 confidence auto-generate explanatory recipes before dispatch.

[quote, README section "Overview" and "Safety Triangle Pipeline"]
____
Hypatia combines: Neural Learning, Symbolic Reasoning, Formal Verification, Graph Intelligence, Distributed Cache, P2P Federation.

The safety triangle pipeline: Eliminate > Substitute > Control
____

*Evidence:* The architecture (lines 49-100) shows all 6 layers implemented:
- Verification (Haskell Liquid types, property testing, formal proofs)
- Learning (neural subsystem with 5 networks)
- Symbolic (Logtalk rules + Prolog inference)
- Data (ArangoDB graph, Dragonfly cache, Radicle P2P)
- Action (robot-repo-automaton, hooks, diagnostics)
- Forge adapters (6: GitHub, GitLab, Bitbucket, Codeberg, sr.ht, Gitea)

The safety triangle (CLAUDE.md lines 53-73) routes findings through:
1. **Eliminate** — Remove root cause permanently (0 risk, lowest cost)
2. **Substitute** — Replace bad component with good alternative (medium cost/risk)
3. **Control** — Add monitoring/limits (highest cost, medium risk, reversible)

This is operationalized in TriangleRouter, FleetDispatcher (confidence-gated), and circuit breaker safety systems.

*Caveat:* The P2P federation (Radicle) is integrated in architecture but not deployed in production; currently uses git-backed verisimdb-data flat files.

Uses the hyperpolymath ABI/FFI standard (Idris2 + Zig). Same pattern across proven, burble, and gossamer.

Hypatia-specific:
